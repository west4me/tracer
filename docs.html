Overview

tracer is a specialized QA tool designed to capture, document, and analyze user interactions within web applications. It provides detailed logging of user actions, accessibility information, and technical context in real-time.

Precision Tracking

Captures every interaction with exact timestamps, element details, and accessibility context.

Smart Documentation

Automatically generates comprehensive logs in multiple formats for easy sharing and integration.

Visual Evidence

Capture screenshots with annotation tools to highlight issues and create visual documentation.

JIRA Integration

Export your test sessions directly to JIRA with customizable fields and Atlassian Wiki markup.

Getting Started

Interface Overview

The tracer interface consists of three main sections:

  • Browser Panel: The main viewing area where you can load websites for testing.
  • Action Log: The right sidebar that records all user interactions.
  • Error Drawer: The bottom panel that logs any JavaScript errors encountered during testing.

Loading a Website

  1. Enter a URL in the address bar at the top of the application.
  2. Click "Load" or press Enter to navigate to the site.
  3. Use the back and refresh buttons to navigate as you would in a normal browser.
Tip: Recently visited sites are saved for quick access on the home page.

Start Recording Actions

  1. Press Ctrl + Space or click the play button in the Action Log panel to start logging.
  2. Interact with the loaded website as you normally would.
  3. Each interaction will be recorded in the Action Log panel with detailed information.
  4. Press Ctrl + Space again to pause logging.
Note: The toggle button's icon will change from to when logging is active.

Interaction Tracking

tracer captures detailed information about various types of user interactions:

Click Events

When you click on an element, tracer records:

  • Element type and content
  • ARIA attributes and accessibility information
  • XPath location
  • Parent container context
  • Element-specific details (e.g., image dimensions, button states)
  • Font information (family, size, weight, style, line height, color)
{ "type": "click", "element": "button.submit-form", "timestamp": "2025-02-11T14:23:42", "context": "checkout-form", "details": { "text": "Submit Order", "aria-label": "Complete purchase", "disabled": false } }

Form Interactions

Form input events capture:

  • Input type and value
  • Validation states
  • Form context
  • Associated labels
  • Radio button groups and checkbox states
{ "type": "input-change", "element": "input[type='email']", "timestamp": "2025-02-11T14:23:45", "details": { "value": "user@example.com", "valid": true, "required": true } }

Navigation Events

Page navigation events track:

  • Page title and URL
  • Load timestamp
  • Previous page information

Keyboard Events

Keyboard interactions record:

  • Key pressed and modifiers (Ctrl, Shift, Alt)
  • Target element
  • Context information

Tab Navigation

Tab navigation events track:

  • Element focus changes when using the Tab key
  • Previous and current elements in focus
  • Details of each element in the tab sequence

Media Interactions

Interactions with media elements (audio, video) capture:

  • Play and pause events
  • Current time and duration
  • Volume changes
  • Media source information

Responsive Testing

tracer includes powerful tools for testing responsive layouts across different device sizes.

Viewport Controls

The viewport toolbar offers multiple ways to test responsive designs:

Desktop View

Default width: 1280px

Tablet View

Default width: 768px

Mobile View

Default width: 375px

Responsive View

Freely resizable with drag handles

Note: The current viewport dimensions are always displayed in the center of the toolbar.

Custom Breakpoints

You can set custom viewport sizes:

  1. Click the settings gear icon in the viewport toolbar
  2. Enter custom width values for Desktop, Tablet, and Mobile views
  3. Click Save to apply your custom breakpoints
Tip: Custom breakpoints are saved between sessions.

Responsive Mode

Test fully responsive layouts with interactive resizing:

  1. Click the responsive view button in the viewport toolbar
  2. Drag the handles on either side to resize the viewport
  3. The current dimensions are displayed in the toolbar

Resetting the Viewport

To return to full-width view, click the reset viewport button in the toolbar.

This will restore the viewport to 100% width of the browser panel.

Tip: Reset to full width when you want to see the entire page without constraints.

Screenshot & Annotation

Capturing Screenshots

tracer offers multiple ways to capture visual evidence:

  • Viewport Screenshot: Press Ctrl + S or click the camera icon on any log entry to capture the current viewport.
  • Element Screenshot: Hold Ctrl and click the camera icon to capture just the specific element.
  • Full Page Screenshot: Hold Ctrl + Shift and click the camera icon to capture the entire page.
Tip: You can take screenshots even when logging is paused.

Annotation Tools

After capturing a screenshot, you can annotate it with the following tools:

Arrow Tool (A)

Draw attention arrows on the screenshot.

Rectangle Tool (R)

Highlight areas with rectangles.

Circle Tool (C)

Draw circles to emphasize points of interest.

Stamp Tool (W)

Add "PASS" or "FAIL" stamps to the screenshot.

Additional Annotation Features:

  • Emoji Picker: Insert emoji annotations
  • Color Picker: Select colors for your annotations
  • Undo/Redo: Fix mistakes with undo and redo buttons

Managing Screenshots

Each screenshot is associated with the log entry where it was captured:

  • Click the screenshot link in a log entry to view and edit the image.
  • Use Undo/Redo to refine your annotations.
  • Click Save to store the annotated image with the log entry.
  • Screenshots are included in exports to provide visual context.
Note: Unsaved annotations will be lost when closing the screenshot view.

Export Formats

Markdown

Clean, readable format perfect for documentation and GitHub.

  • Structured text with sections
  • Code blocks for technical details
  • Image references
  • Compatible with most documentation tools

HTML Report

Rich, interactive reports with full detail and screenshots.

  • Interactive timeline
  • Embedded screenshots
  • Expandable details
  • Styled for readability

JIRA Format

Ready-to-paste format for JIRA tickets and comments.

  • Atlassian Wiki markup
  • Customizable JIRA fields
  • CSV Import format
  • Organized sections with panels

Using Exports

To export your action log:

  1. Click one of the export buttons in the Action Log toolbar:
    • Export as Markdown
    • Export as HTML
    • Export as JIRA
  2. For JIRA exports, you can customize fields and issue type (see JIRA Integration section)
  3. The exported file will be saved to your downloads folder
Tip: You can export just the comments as a JIRA comment by holding Ctrl while clicking the JIRA export button.

Commenting System

tracer allows you to add rich-text comments to any recorded event, providing context and explanations for your test steps.

Adding Comments

To add a comment to any log entry:

  1. Click the comment icon on the event you want to annotate
  2. Type your comment in the text editor
  3. Format your text using the toolbar options
  4. Click "Save" to attach the comment to the event

Text Formatting Options

B Bold Text

Format text as bold for emphasis.

U Underline

Add underline formatting to text.

List Bullet List

Create bulleted lists for multiple items.

Link Insert Link

Add hyperlinks to external resources.

Managing Comments

  • Edit a comment by clicking the comment text and then the edit button
  • Delete comments using the delete button on each comment
  • All comments are included when exporting to Markdown, HTML, or JIRA formats
Tip: Use Ctrl + Enter to save a comment quickly.

JIRA Integration

tracer provides advanced JIRA integration to streamline your bug reporting workflow.

Export to JIRA

Export your test session as a JIRA-ready CSV file:

  1. Click the JIRA export button in the Action Log toolbar
  2. Configure the export settings in the dialog:
    • Set the Issue Type (Defect, Story, Task, etc.)
    • Enter a Summary for the issue
    • Add custom JIRA fields as needed
  3. Click "Save & Export" to generate the CSV file
  4. Import the CSV into JIRA using the CSV import feature

SharePoint/OneDrive Integration

For screenshots in JIRA exports:

  1. When exporting to JIRA with screenshots, you'll be prompted to save them to a folder
  2. Choose a folder in your SharePoint/OneDrive directory
  3. Enter your SharePoint username when prompted
  4. The export will include proper SharePoint links to your screenshots
Note: Using SharePoint/OneDrive ensures proper image embedding in JIRA tickets.

Atlassian Wiki Markup

tracer formats your logs with Atlassian Wiki markup for rich formatting in JIRA:

  • Structured headings for each section
  • Formatted panels for different types of information
  • Code blocks for technical details
  • Image references for screenshots
  • Bulleted and numbered lists
  • Color-coded text for highlighting important information

JIRA Comment Export

You can also export just the comments section for quick JIRA updates:

  1. Hold Ctrl while clicking the JIRA export button
  2. This opens a dialog with just the comments formatted as JIRA Wiki markup
  3. Copy the text and paste it directly into a JIRA comment

Custom Fields

Add custom JIRA fields to your export:

  • Component, Priority, Labels, Assignee, etc.
  • Recently used fields are saved for quick access
  • Fields are validated to ensure proper JIRA import
Note: Custom fields must match your JIRA instance configuration. Consult your JIRA administrator for available fields.

Error Logging

tracer automatically captures JavaScript errors that occur during testing, helping you identify issues in the application.

Error Drawer

JavaScript errors are tracked in the Error Drawer at the bottom of the screen:

  • When errors occur, the error count appears on the error drawer button
  • The button turns red to indicate errors are present
  • Click the button to open the Error Drawer and view detailed error information

Error Details

Each error entry includes:

  • Error type and message
  • Source file and line number
  • Stack trace for debugging
  • Timestamp of when the error occurred

Managing Errors

Copy Error Details

Copy error information to clipboard with the copy button.

Export Errors

Export the entire error log to a text file.

Clear Errors

Clear all errors from the log.

Toggle Drawer

Hide/show the error drawer.

Error Filtering

tracer automatically filters out common non-critical errors to reduce noise:

  • Console warnings and info messages
  • Navigation aborted errors
  • Standard browser notification messages

This ensures the error log contains only meaningful issues that might affect application behavior.

Tip: The error log automatically clears when you refresh the page.

Finding Text

tracer includes a powerful find-in-page feature to help you locate specific text within the current webpage.

Using Find in Page

To search for text within the current page:

  1. Press Ctrl + F to open the find bar
  2. Type your search query in the input field
  3. Results will be highlighted in the page as you type
  4. Press Esc or click the X button to close the find bar

Find Bar Features

The find bar provides real-time feedback as you type. Results highlight in the page, the input border turns red if no matches are found, and you can click outside the find bar to dismiss it.

Tip: You can press Enter to cycle through multiple matches on the page.
